# 📖 主线剧情与任务设计文档

## 📋 文档说明

本文档详细记录了《无尽修仙》游戏的主线剧情、主线任务模板系统、每日任务系统的设计与实现。

**最后更新**: 2026-03-22
**版本**: v2.1
**维护者**: 游戏设计团队

## 🎮 核心世界观

### 背景设定

在遥远的极北之地，一个被永恒寒冬笼罩的世界里，修仙者们为了突破境界、追求永生而不断奋斗。传说中，只有通过层层考验，最终达到化神境的强者，才能打破这个世界的枷锁，找到通往更高维度的道路。

### 世界观元素

- **永恒寒冬**：世界被冰雪覆盖，资源匮乏，生存环境恶劣
- **境界体系**：武者境 → 炼气境 → 筑基境 → 金丹境 → 元婴境 → 化神境
- **修仙门派**：各大门派为争夺资源和传承而竞争
- **秘境挑战**：不同境界的修仙者可以进入对应的秘境
- **天道考验**：突破境界时需要面对的考验

## 📜 主线剧情（已实现）

### 剧情结构

采用 **里程碑剧情 + 日志叙事** 双层结构：
- **里程碑剧情**：全屏覆盖层剧情演出，仅在关键节点触发（境界开始、阶段过渡、Boss 战、境界圆满）
- **日志叙事**：模板化的叙事文字，在战斗日志中显示，每个任务完成时触发

### 六卷剧情

| 卷 | 境界 | 剧情标题 | 伴随角色 | 核心目标 |
|---|------|---------|---------|---------|
| 第一卷 | 武者境 | 武者之路 | 师尊 | 突破武者境的极限 |
| 第二卷 | 炼气境 | 炼气有成 | 师兄 | 筑基凝真，踏入修仙正途 |
| 第三卷 | 筑基境 | 筑基求真 | 弟子 | 结成金丹，大道可期 |
| 第四卷 | 金丹境 | 金丹大道 | 长老 | 碎丹成婴，超凡脱俗 |
| 第五卷 | 元婴境 | 元婴无双 | 老祖 | 元神合道，化神可期 |
| 第六卷 | 化神境 | 化神超脱 | 天道 | 打破永恒寒冬，飞升成仙 |

### 里程碑触发点

| 触发类型 | 触发时机 | 数量 |
|---------|---------|------|
| 境界开始 (`chapter_start`) | 进入新境界时 | 6 个 |
| 阶段过渡 (`stage_transition`) | 第 4/7/10 阶段完成时 | 6 个境界 × 3 = 18 个 |
| Boss 战 (`boss_battle`) | Boss 击杀剧情 | 12 个 |
| 境界圆满 (`realm_final`) | 境界突破完成时 | 6 个 |

**总计**：42 个里程碑剧情场景

### 剧情文件位置

- 场景定义：`game-metadata.js` → `storyScenes.scenes`
- 剧情引擎：`mainQuest.js` → `triggerStory()`, `showStoryOverlay()`, `nextStoryPage()`
- 剧情 UI：`index.html` → `#story-overlay`

## 🎯 主线任务系统（已实现 - 模板系统 v2.0）

### 设计目标

替代原有的静态 60 任务方案，使用模板动态生成，确保每个境界每个阶段的每个等级都有任务覆盖。

### 核心机制

```
境界 (Realm, 0-5) → 阶段 (Stage, 1-10) → 等级 (Level, 1-levelCap)
                          ↓
              每个等级生成 2 个任务（从模板池动态选取）
```

### 任务模板类型（8 种）

| 模板类型 | 说明 | 目标公式 |
|---------|------|---------|
| `kill_normal` | 击杀普通怪物 | `baseKill × stageMult × (1 + level × 0.1)` |
| `kill_elite` | 击杀精英怪物 | `ceil(baseKill × stageMult × 0.6 × (1 + level × 0.1))` |
| `kill_boss` | 击杀 Boss | 固定 1 |
| `collect_herbs` | 收集灵草 | `baseCollect × stageMult × (1 + level × 0.1)` |
| `collect_iron` | 收集玄铁 | `baseCollect × stageMult × (1 + level × 0.1)` |
| `collect_spiritStones` | 收集灵石 | `baseCollect × stageMult × 0.8 × (1 + level × 0.1)` |
| `visit_map` | 前往指定地图 | 固定 1 |
| `reach_level` | 达到目标等级 | 固定 1 |

### 数值缩放公式

```javascript
// 阶段乘数（Stage 1-10）
stageMultiplier: [1.0, 1.3, 1.6, 2.0, 2.4, 2.8, 3.3, 3.8, 4.3, 5.0]

// 境界乘数（Realm 0-5）
realmMultiplier: [1.0, 1.5, 2.2, 3.2, 4.5, 6.5]

// 基础数值
baseKill: 5, baseCollect: 8, baseExp: 80, baseSpiritStones: 40

// 击杀目标 = baseKill × stageMult × (1 + level × 0.1)
// 经验奖励 = baseExp × stageMult × realmMult × (1 + level × 0.05)
// 灵石奖励 = baseSpiritStones × stageMult × realmMult × (1 + level × 0.05)
```

### 武者境缩放示例

| Stage | Level | killTarget | collectTarget | expReward | spiritStonesReward |
|-------|-------|-----------|---------------|-----------|-------------------|
| 1 | 1 | 5 | 8 | 80 | 40 |
| 1 | 10 | 10 | 16 | 120 | 60 |
| 5 | 1 | 12 | 19 | 192 | 96 |
| 5 | 10 | 24 | 38 | 288 | 144 |
| 10 | 1 | 25 | 40 | 400 | 200 |
| 10 | 10 | 50 | 80 | 600 | 300 |

### 任务选取规则

每个等级从模板池中按以下规则选取 2 个不重复的任务：

| 条件 | 选取模板 |
|------|---------|
| 每个阶段的第 1 级 | `visit_map` + `kill_normal` |
| 每 5 级 | `kill_boss` + `kill_elite` |
| 奇数级（非上述） | `kill_normal` + `collect_*` |
| 偶数级（非上述） | `kill_elite` + `collect_*` |

### 境界主题配置

每个境界有独立的地图、Boss 池、资源和伴随角色，驱动任务生成：

| 境界 | 地图 | Boss 池 | 主要资源 | 伴随角色 |
|------|------|---------|---------|---------|
| 武者 | 仙侠山峰 | 冰霜巨人 | 灵草 + 玄铁 | 师尊 |
| 炼气 | 仙侠山峰 + 仙侠海滩 | 龙王、海怪 | 灵草 + 玄铁 | 师兄 |
| 筑基 | 仙侠海滩 + 仙侠平原 | 草原之王 | 玄铁 + 灵草 | 弟子 |
| 金丹 | 仙侠峡谷 + 仙侠沙漠 | 峡谷领主、沙漠之王 | 灵石 + 玄铁 | 长老 |
| 元婴 | 仙侠湖泊 + 仙侠森林 + 仙侠火山 | 湖龙王、妖狐王、火山领主 | 灵石 + 玄铁 | 老祖 |
| 化神 | 仙侠洞穴 + 仙侠仙境 | 元始天尊 | 灵石 + 玄铁 | 天道 |

## 📅 每日任务系统（已实现）

### 系统概述

每日 3 个随机任务，每天凌晨 0 点自动刷新，提供经验 + 灵石 + 活跃度点数奖励，支持连续完成加成。

### 每日任务模板池（7 种，每天随机选 3 个不重复）

| 模板 ID | 名称 | 目标类型 | 目标公式 |
|---------|------|---------|---------|
| `daily_kill_normal` | 妖兽讨伐 | 击杀普通怪物 | `max(3, baseKill × stageMult)` |
| `daily_kill_elite` | 精英猎杀 | 击杀精英怪物 | `max(3, baseKill × stageMult)` |
| `daily_kill_boss` | Boss 挑战 | 击杀 Boss | `max(1, baseBoss × stageMult)` |
| `daily_collect_herbs` | 灵草收集 | 收集灵草 | `max(3, baseCollect × stageMult)` |
| `daily_collect_iron` | 玄铁收集 | 收集玄铁 | `max(3, baseCollect × stageMult)` |
| `daily_collect_stones` | 灵石收集 | 收集灵石 | `max(3, baseCollect × stageMult)` |
| `daily_visit_map` | 地图探索 | 前往地图 | 固定 1 |

### 每日任务基础数值

```
kill: 10, collect: 12, boss: 1, exp: 60, spiritStones: 30, activity: 10
```

### 每日任务缩放

```javascript
// 复用主线模板的 stageMultiplier 和 realmMultiplier
target = baseValue × stageMult
expReward = 60 × stageMult × realmMult
spiritStonesReward = 30 × stageMult × realmMult
activityPoints = 10 × stageMult × realmMult
```

### 连续完成奖励

| 连续天数 | 奖励加成 | 额外活跃度 | 称号 |
|---------|---------|-----------|------|
| 3 天 | +50% | +30 | 小有成就 |
| 5 天 | +80% | +60 | 初窥门径 |
| 7 天 | +100% | +100 | 七日修炼 |
| 14 天 | +150% | +200 | 半月精进 |
| 30 天 | +200% | +500 | 一月大成 |

### 状态结构

```javascript
gameState.dailyQuests = {
    lastRefreshDate: "2026-03-14",  // 上次刷新日期
    quests: [/* 3个每日任务 */],
    activityPoints: 0,              // 累计活跃度
    streak: 0,                      // 连续完成天数
    totalCompleted: 0,              // 累计完成数
    completedToday: false           // 今日是否全部完成
}
```

### 活跃度用途（后续扩展）

| 活跃度 | 兑换物品 |
|--------|---------|
| 30 点 | 突破石 ×3 |
| 100 点 | 稀有材料包 |
| 200 点 | 蓝色装备箱 |
| 500 点 | 紫色装备箱 |
| 1000 点 | 化神秘宝箱 |

### 实现文件

| 文件 | 职责 |
|------|------|
| `game-metadata.js` → `dailyQuestConfig` | 模板池、基础数值、连续奖励配置 |
| `dailyQuest.js` | 生成、刷新、进度追踪、奖励领取、UI 渲染 |
